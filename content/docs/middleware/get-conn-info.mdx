The `getConnInfo` middleware extracts connection details (`IP`, `port`, `protocol` family) from the `Bun` server and attaches them to `ctx.req.remoteAddress`. This allows any route or middleware to easily access the client’s network information.

## Import

```ts
import { getConnInfo } from "tezx/middleware";
```

---

## Purpose

`getConnInfo` provides a unified way to access client connection metadata directly from the request context.
This is useful for:

* Logging client IPs
* Rate limiting
* Bot detection
* Security and fraud checks
* Geo-based handling

---

## Usage Example

```ts
import { getConnInfo } from "tezx/middleware";

app.use(getConnInfo());

router.get("/", (ctx) => {
  const ip = ctx.req.remoteAddress?.address;
  return new Response(`Your IP: ${ip}`);
});
```

---

## What It Does

* Reads the connection information using Bun’s native `server.requestIP(rawRequest)`
* Injects the result into `ctx.req.remoteAddress`
* Continues request execution via `next()`

No transformation, no parsing — minimal overhead.

---

## API Reference

### `getConnInfo(): Middleware`

Returns a middleware function that enriches the context with connection metadata.

#### Injected Structure (`NetAddr`)

```ts
type NetAddr = {
  address: string;  // client IP
  port: number;     // client port
  family: "IPv4" | "IPv6";
};
```

### Behavior

1. Checks if the server supports `requestIP`.
2. Calls `server.requestIP(ctx.rawRequest)` to get the client address.
3. Stores the result in `ctx.req.remoteAddress`.
